## 測試前的準備

在撰寫測試前，首先需要明確元件的功能和使用情境。舉例來說，針對一個登入表單，測試案例可能會如下：

```tsx
describe("登入表單", () => {
  // 正確路徑（Happy Path）
  test("輸入正確的帳號和密碼，應登入後台", () => {
    // 測試內容
  });
  // 錯誤路徑（Sad Path）
  test("僅輸入帳號，應顯示「請輸入密碼」", () => {
    // 測試內容
  });
  test("僅輸入密碼，應顯示「請輸入帳號」", () => {
    // 測試內容
  });
  test("輸入錯誤的帳號或密碼，應顯示「登入資訊錯誤」", () => {
    // 測試內容
  });
});
```

## 基礎語法解析

### `describe`

作用：將多個 test 分組和命名。
說明：透過 describe 可以提供共享的作用域和描述。

### `test` 或 `it`

作用：定義單一測試案例。
說明：每個 test 應獨立測試一項特定的功能或情況。
建議：使用 test 因為它更為直觀。

### 斷言（Assertions）

`expect`
作用：建立一個斷言。
說明：用於檢查某個條件是否符合預期。

### 匹配器（Matchers）

`toBe`
作用：用於比較基本數據類型或對象引用。
說明：確保值或對象引用與預期相同。
`toEqual`
作用：進行深度比較。
說明：確保對象或陣列在結構和值上都相等。
